    Multi-Domain Mirasys Setup

    This guide helps you configure the Mirasys application with support for multiple custom domain names and ports.

    1. Get the Project Files

    Clone the project repository:

    git clone <project_repository_link>

    2. Update Configuration (app.env.dev)

    Update the following keys in your app.env.dev file to reflect your custom domain and port setup:

    # Environment configuration for Docker Compose
#=============================================

# Service URLs

    3. Edit Docker Configuration (docker-compose.local.yaml)

    Update the container names and port settings:

    version: "4.2.5"

# Define a reusable network configuration
x-mirasys-network: &mirasys-network
  networks:
    - mirasys-network

services:
  {{C_PREFIX}}_traefik:
    image: traefik:v3.0.0-beta5
    container_name: {{C_PREFIX}}_traefik_mirasys
    restart: unless-stopped
    command:
      - "--log.level=DEBUG"
      - "--api.insecure=true"
      - "--providers.docker=true"
      - "--providers.docker.exposedbydefault=false"
    ports:
      - "{{STATIC_PORT}}:80"
      - "{{TRAEFIK_PORT}}:8080"
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock
    depends_on:
      - "phpmyadmin"
      - "{{C_PREFIX}}_infra"
      - "{{C_PREFIX}}_mirasys_api_gateway"
      - "{{C_PREFIX}}_mirasys_frontend_central"
      - "{{C_PREFIX}}_mirasys_frnt"
      - "{{C_PREFIX}}_mirasys_backend"
      - "rabbitmq"
      - "{{C_PREFIX}}_mirasys_streaming"
      - "{{C_PREFIX}}_mirasys_api_tenant_gateway"
    labels:
      - "traefik.enable=true"
      - "traefik.http.routers.{{C_PREFIX}}_traefik.rule=Host(`${TRAEFIK_URL}`)"
      - "traefik.http.routers.{{C_PREFIX}}_traefik.service=api@internal"
      - "traefik.http.routers.{{C_PREFIX}}_traefik.middlewares=traefik-auth"
      - "traefik.http.middlewares.traefik-auth.basicauth.users=systemteam:$2a$10$1kocyz5oVBUYlojfFDatPeYgpyx8yM6ATVj/W2yE/E/fLTlRlXYSe"

  db:

    Note: Make sure to replace {{C_PREFIX}} with your desired prefix for the containers. Ensure that the docker-compose.dev.yaml file is correctly configured with the necessary environment variables and paths.

    4. Deploy the Project

    sudo ./deploy.sh init-images && \
sudo ./deploy.sh development

    5. Configure the Host File

    sudo nano /etc/hosts

    Add the following:

    127.0.0.1 phpmyadmin.demo.mirasys.in
127.0.0.1 frontend-central.demo.mirasys.in
127.0.0.1 demo.mirasys.in
127.0.0.1 api-gateway.demo.mirasys.in
127.0.0.1 tenant-api-gateway.demo.mirasys.in
127.0.0.1 backend.demo.mirasys.in
127.0.0.1 stream.demo.mirasys.in
127.0.0.1 traefik.demo.mirasys.in
127.0.0.1 rabbitmq.demo.mirasys.in
127.0.0.1 infra.demo.mirasys.in

    Press CTRL+X, then Y, then ENTER to save.

    6. Set Up the Database

    sudo ./create-tenant-db.sh

    ✅ Setup Completed!

    Your Mirasys system is now ready with multiple domain support.
