    Manual Installation

    1. Clean Previous Configurations (Optional)

    rm -f .env docker-compose.yaml

    2. Initialize Docker Images

    sudo ./deploy.sh init-images

    3. Build and Deploy the Application

    sudo ./deploy.sh development

    4. Host File Configuration

    Edit the host file to map service hostnames to localhost:

    sudo nano /etc/hosts

    Append the following entries:

    127.0.0.1 phpmyadmin_mirasys.local
127.0.0.1 mirasys_frontend_central.local
127.0.0.1 mirasys_frnt.local
127.0.0.1 mirasys_api_gateway.local
127.0.0.1 mirasys_api_tenant_gateway.local
127.0.0.1 mirasys_backend.local
127.0.0.1 mirasys_stream.local
127.0.0.1 traefik_mirasys.local
127.0.0.1 rabbitmq.local
127.0.0.1 infra.local

    To save and exit:

    • Press Ctrl + X, then Y, and hit Enter.

    5. Database Initialization

    ./create-tenant-db.sh

    Script Functionality Overview

    Function Description
    ls -lah /var/lib/mysql Check existing MySQL storage
    chown -R mysql:mysql /var/lib/mysql Ensure MySQL file ownership
    chmod -R 755 /var/lib/mysql Set appropriate permissions
    Drop mirasys, mirasys_tenant databases Clean start
    Create fresh databases Prepares empty DBs
    Run Sequelize migrations Applies schema
    Seed databases Loads default data
    Import SQL schema Setup table structure
    Additional tenant migrations and seeds Enable multi-tenancy
    Install dependencies (jq, curl) Required tools for API interaction
    Login to GraphQL API Retrieve access token
    Send authenticated alert schema setup Initializes alert system

    6. Accessing the Application Dashboard

    Open your browser and navigate to:

    http://mirasys_frnt.local/

    7. Troubleshooting

    Issue Solution
    Docker command fails Ensure user is in the docker group
    DB migration errors Re-run create-tenant-db.sh and check MySQL logs
    Services not resolving Re-check /etc/hosts configuration
